Project information

  • Category: Web development, Native development
  • Client: Invisible Urban
  • Project URL: IU Console
  • App URL: IU Charging App
  • Project Description: A EV charging system includes a mobile app, web admin console, API, and WebSocket servers. Customers can initiate a charging session by scanning a QR code through the mobile app, which then sends a request to the API server. This request triggers another request to the WebSocket server, which facilitates communication between the charging station and the API server.
  • Technologies: LaravelVue

My responsibilities

  • Created APIs based on a specific charger protocol to ensure seamless communication between the charging station and the EV charging system.
  • Developed a simulator that can replicate the actions of an actual charger, allowing for rigorous testing of the system in a controlled environment.
  • Designed and implemented an admin console that includes management features for various aspects of the system, such as site, station, connector, account, and user management.
  • Effectively communicated with customers and provided support during real device testing, ensuring that any issues were identified and addressed in a timely manner.
